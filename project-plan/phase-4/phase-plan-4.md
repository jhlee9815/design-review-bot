# Phase 4 Plan — 대표 컴포넌트 적용 실험

## 목표

Apple-inspired DS를 전체 앱에 적용하지 않고, 대표 컴포넌트 2~3개만 선정해 작은 end-to-end 적용 증거를 만든다.

## 추천 대상

1. Button
2. Card
3. Hero 또는 Navigation

## 작업 절차

1. 적용 대상 컴포넌트 선정
2. 기존 UNO 스타일과 Apple token adapter 비교
3. 최소 변경으로 variant 또는 demo section 추가
4. build/lint 실행
5. 필요 시 시각 확인 자료 생성

## 산출물

- `project-plan/phase-4/phase-plan-4.md`
- 적용 컴포넌트 변경 내역
- before/after 기록

## 완료 조건

- 최소 2개 컴포넌트에서 Apple-inspired 규칙 적용이 확인됨
- build/lint 통과
- 자동 적용/사람 검토 경계가 설명 가능함
